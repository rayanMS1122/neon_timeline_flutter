# Release Report 4.0 — Advanced Foundation

## Decision

**Not approved for publication yet.**

This worktree is a substantial engineering and UI upgrade over the uploaded
4.0 foundation, but publication remains blocked until the real Flutter quality
gate and documented performance measurements pass.

## Corrected defects

- Replaced repeated conflict scans with O(1) conflict membership/type indexes.
- Replaced potentially quadratic conflict-cluster classification with a linear
  cluster pass.
- Corrected selected-day calculation for calendar days affected by daylight
  saving changes.
- Prevented stale render plans when entry values or in-place lists change.
- Reworked capacity analysis from resource×entry scanning to assignment
  indexing plus per-resource event sweeps.
- Removed per-card capacity-conflict rescans.
- Corrected the benchmark's reference to a nonexistent API.
- Made neutral card/panel backdrop blur explicit opt-in.
- Hardened tiny viewport and geometry bounds.
- Removed the example's double mobile bottom navigation.
- Corrected stale example tests that still expected the old 4.0 placeholder.
- Fixed CI artifact validation so it does not reject `.dart_tool` generated by
  its own `flutter pub get` step.
- Corrected analytics so off-window entries no longer affect counts, completion,
  resource totals, or scheduled duration.
- Prevented an invalid range from assigning `invalidRange` to valid entries in
  the same overlap cluster.
- Added latest starts, slack, exact critical-entry sets, and exact critical-edge
  sets to dependency analysis.
- Rejected duplicate entry and dependency IDs instead of silently collapsing
  ambiguous project graphs.
- Added a synchronized time header to the resource board.
- Removed fixed package-owned UI strings from advanced fallbacks by introducing
  host-injected `TimelineLocalizationData` without an `intl` dependency.
- Guarded `TimelineCommandHistory` against late async completion after disposal,
  including stack mutation and listener notification.

## New production UI

- `CalendarDayView<T>`
- `ResourceTimelineView<T>`
- `DependencyTimelineView<T>`
- `TimelineWorkspace`
- modern dashboard panels, metrics, badges, headers, and backdrop
- Aurora and Soft Professional design systems
- a full interactive Timeline Studio example with dashboard, day, resources,
  planner, roadmap, agenda, dependency graph, filters, inspector, and themes

## New logic and diagnostics

- deterministic day overlap layout;
- schedule analytics including range-clipped occupancy, utilization,
  concurrency, completion, and resource counts;
- richer controller state for focus, range selection, zoom, panning, and batched
  notifications;
- adaptive performance resolution for web, dataset size, low power, and reduced
  motion;
- CPM-style dependency timing with earliest/latest starts and slack;
- stronger release scripts and multi-platform CI, including Android and web
  release builds in the primary quality gate.
