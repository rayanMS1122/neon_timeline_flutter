# Neon Timeline Flutter — Example App

Comprehensive interactive demo showcasing **every public API** of the `neon_timeline_flutter` package.

## Quick Start

```bash
cd example
flutter pub get
flutter run
```

### Android

```bash
cd example
flutter run -d android
```

### iOS

```bash
cd example
flutter run -d ios
```

### Web

```bash
cd example
flutter run -d chrome
```

### Tests

```bash
cd example
flutter test
```

### Analyze

```bash
cd example
flutter analyze
```

## Key Implementation Details

### Fake Data (No External Deps)
All demo data is generated in-memory via `FakeRepository` in `demo_data.dart`:
- `DemoTask` — schedule entries with title, subtitle, time, duration, color, status
- `DemoTimelineItem` — generic timeline items with content/opposite/indicator
- Deterministic `Random(42)` for reproducible layouts

### Theme System
The app wraps everything in `NeonTimelineTheme` with a runtime-swappable `NeonTimelineThemeData`. All 15 presets are available via dropdown.

### Performance on Web
- `NeonBlurCache` returns `null` MaskFilters on Web
- `NeonPaintBlur.applyBlur()` becomes no-op
- Continuous animations disabled; only hover/focus transitions
- FPS capped at 30 by default

### Accessibility
- Every interactive element has `semanticLabel`
- `FocusableActionDetector` + `ActivateIntent` for keyboard activation
- Minimum 48×48dp touch targets
- Respects `MediaQuery.disableAnimations`

### Testing
The **Test Runner** screen executes 6 real `flutter_test` widget tests in-process and displays results live. Tests cover:
1. Timeline item rendering
2. Schedule drag-to-reschedule
3. Slide action callbacks
4. Day pager swipe navigation
5. Theme preset application
6. Motion scope reduce-motion behavior

## Troubleshooting

| Issue | Fix |
|-------|-----|
| `flutter analyze` shows deprecated `withOpacity` | Use `.withValues(alpha: x)` — info-only, not blocking |
| Web build slow / high CPU | Ensure `kIsWeb` branch disables blur; reduce `motionFramesPerSecond` |
| Android build fails | `flutter clean && flutter pub get && flutter build apk` |
| Test runner shows "Not run" | Tap individual **Run** button or **Run All** |
